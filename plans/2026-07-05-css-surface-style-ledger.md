## Boundary Rules

- `surgeist-css` owns strict parsing, CSS syntax recovery policy, CSS source
  locations, and authored CSS syntax types.
- Root owns lowering from `surgeist-css` types into style front-door APIs and
  integration diagnostics for unsupported CSS.
- `surgeist-style` owns typed receiving models, cascade semantics, selector
  matching contracts over root-provided facts, inheritance, invalidation, and
  computed style outputs.
- `surgeist-style` must not depend on `surgeist-css`.
- Every CSS surface item in this ledger is classified as `Existing style model`,
  `New style model needed`, `Typed symbolic style data`, `Root rejection`,
  `Out of style boundary`, or `Root-owned lowering boundary`.

## Classification Labels

| Label | Meaning |
| --- | --- |
| `Existing style model` | Style already has a typed model or behavior that can receive this surface intentionally. |
| `New style model needed` | Style owns the semantic domain, but a new typed model or behavior must be added. |
| `Typed symbolic style data` | Style should preserve typed symbolic data until a later owner or context can resolve it. |
| `Root rejection` | Root should reject this parsed CSS surface for now with an unsupported-integration diagnostic. |
| `Out of style boundary` | The surface is parsed by CSS but belongs to another crate or host concern, not style. |
| `Root-owned lowering boundary` | Root must translate this CSS syntax into style-owned inputs; style must not import CSS types. |

## Current Style Surface Summary

| Style surface | Current role | Gap against CSS surface |
| --- | --- | --- |
| `Property` | Non-exhaustive style-owned property enum with canonical/shorthand metadata, defaults, inheritance, interpolation, and invalidation impact. | Covers the existing Rust style property set, but does not cover the full `CssProperty` ledger, custom properties, CSS cascade metadata, or several CSS longhands and shorthands accepted by CSS. |
| `Value` | Typed style value enum used by declarations, validation, interpolation, hashing, and resolved outputs. | Represents many non-variable layout, text, paint, and effect values, but lacks authored CSS token preservation, variable-dependent values, custom property values, `revert`/`revert-layer`, and many CSS value families recorded as new style models. |
| `Declaration` | Validated `Property` plus `Value` pair for style-owned declarations. | Does not carry CSS source location, importance, origin, layer, specificity, custom property names, or variable-dependent authored values needed for CSS lowering and cascade. |
| `Declarations` | Canonicalized collection of declarations with replacement semantics, shorthand expansion, lookup, iteration, and fingerprinting. | Loses authored declaration ordering and duplicates by canonical property, and lacks CSS cascade buckets for `!important`, custom properties, CSS-wide keywords beyond current resolution, and variable substitution state. |
| `TypedDeclaration` | Small typed front-door constructor wrapper for selected Rust-authored declarations such as width, opacity, text color, and transition duration. | Far narrower than the CSS declaration surface; CSS lowering still needs a comprehensive typed receiving API across all supported properties and symbolic authored values. |
| `Sheet` | Ordered rule store with selector indexing, versioning, conditional-rule iteration, candidate lookup, and condition-driven invalidation helpers. | Does not model CSS stylesheet-level constructs such as imports, named layers, font-face blocks, keyframes, scoped rules, media/container query lists, origins, or source-order groups beyond a flat rule order. |
| `Rule` | Style rule containing one selector, one declaration set, optional simple conditions, and flat order. | Lacks CSS selector lists, specificity, cascade origin, layer, scope roots/limits, `!important` separation, source locations, pseudo-element targets, and nested at-rule context. |
| `Selector` | Style-owned matcher over tags, classes, keys, states, attributes, positions, compounds, and complex parts against `Tree` facts. | Does not cover full CSS selector syntax including selector lists, namespaces, richer attribute operators/flags, specificity accounting, pseudo-elements, `:not()`/`:is()`/`:where()`/`:has()`, type-filtered structural selectors, or relative selectors. |
| `Tree` | Root-provided trait exposing node facts and canonical/projected traversal for selector matching and resolver cache keys. | Provides only the current style fact contract; CSS matching needs additional facts or policies for namespaces, language/direction, pseudo-element targets, empty/text semantics, type-filtered sibling matching, shadow/slot boundaries, and `:has()` dependency tracking. |
| `Context` | Resolver input carrying tree/node, traversal mode, viewport, optional container, parent resolved style, local declarations, and animated declarations. | Lacks CSS cascade origin/layer inputs, environment/media facts beyond width/height helpers, custom property context, scoped-rule state, pseudo-target selection, and root lowering diagnostics. |
| `Resolver` | Applies candidate sheet rules, conditions, inheritance, local declarations, animated declarations, keyword resolution, and cache management to produce `Resolved`. | Current cascade is flat and style-authored; CSS requires specificity, origins, layers, importance, scoped proximity, media/container query lists, custom property substitution, `revert`/`revert-layer`, and richer invalidation dependencies. |
| `Resolved` | Computed style map storing every canonical property with default/inherited values and typed convenience accessors. | Does not expose computed custom properties, unresolved symbolic color/value data, pseudo-element resolved targets, layer/origin provenance, or CSS property families still absent from `Property`/`Value`. |

| `Change` | Rematch plus invalidation and node/sibling/descendant scope summary derived from property changes or conditions. | Lacks CSS-specific change reasons for class/attribute/state selector dependencies, structural selector dependencies, `:has()` ancestor effects, custom property substitutions, media/container query feature changes, and layer/scope order changes. |
| `Condition` | Simple conjunction over viewport and optional container width/height query helpers. | Does not model CSS media query lists, media types/modifiers, n-ary `and`/`or`/`not`, feature ranges beyond ad hoc width/height min/max, named containers, style queries, scope, or environment preferences. |
